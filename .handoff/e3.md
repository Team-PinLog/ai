## 4. 확정된 판단·근거 (바꾸지 말 것)

- **Fake만 사용** — 실 GMS API 호출 테스트 없음(비용·비결정성). `FakeEmbeddingClient`/`FakeLLMClient`.
- **호출 횟수 단언이 핵심** — 시나리오 1·5·6·7·9·13·16·18은 "호출 안 함(`call_count==0`)"/"정확히 한 번" 단언.
- **동시성은 `on_call` 훅으로 순서 고정, `sleep` 금지** — 시나리오 9·10·11·12·1·2는 `asyncio.gather` + Fake `on_call`에서 다른 커넥션(`raw_connect`)으로 State를 CANCELLED 변경. 모델 호출과 저장 사이 창을 결정론적으로 재현.
- **저장 불변식** — 저장 직전 `FOR UPDATE` 재검사(`embedding_status=='PROCESSING'` / keyword는 `embedding=='COMPLETED' AND keyword=='PROCESSING'`). CANCELLED면 rowcount 0으로 폐기(예외 아님).
- **격리는 TRUNCATE**(롤백 격리 불가 — 멀티 커넥션). conftest `db` fixture가 함수별 TRUNCATE.
- **Profile 문자열 리터럴 금지** — `settings` fixture 경유.
- **builders에 본문 버전 인자 없음** — 수정 시나리오(1)는 `context_id` 다른 두 State로.
- 환경 고정: **pgvector 0.8.1-pg16**, **Python 3.12(<3.13)**.
